六月九日下午，Hacker News 首頁被一篇標題幾乎佔滿的新聞壓制：

> **Claude Fable 5 (anthropic.com) — 1728 points, 1365 comments**

這是 Anthropic 第一次把「Mythos 等級」的模型公開釋出。Artificial Analysis 的獨立基準測試顯示，Fable 5 在衡量真實世界知識工作代理的 GDPval-AA 拿到 **1932 分**——穩坐第一，把 Anthropic 整個實驗室推上「前四名佔三席」的位置。模型用 $10 / 百萬輸入 token 與 $50 / 百萬輸出 token 的價格（剛好是 Opus 4.8 的兩倍、Mythos Preview 的一半），透過 Claude API、AWS Bedrock、Vertex AI、Microsoft Foundry 同步開賣。

任何寫過 prompt、跑過 benchmark、付過 API 帳單的人——任何有眼睛的人——都會同意：這是史上最強的公開模型。

但模型本身只佔了故事的一半。另一半，是它**怎麼被閹掉**的。

## 一、雙重標準：看得見的降級 vs. 看不見的降級

Anthropic 在 Fable 5 同步推出一組新的「安全分類器（safety classifiers）」，目的是攔截高風險領域的請求。規則分成兩層，**執行的方式完全不同**。

**第一層：可見的降級——資安、生物、化學、蒸餾攻擊**

當使用者的請求觸發這些領域的分類器時，Anthropic 會**明確告知使用者**「這個請求無法用 Fable 5 處理」，然後把請求**降級到 Claude Opus 4.8** 來回應。Anthropic 在自家部落格說：「對使用者而言，被降級到 Opus 是一個遠好過直接拒絕的體驗。早期數據顯示，**95% 以上的 Fable session 完全不觸發降級**——這些 session 裡，Fable 5 的表現和 Mythos 5 沒有差異。」

這套機制可以辯護：風險領域明確、機制透明、行為可預測。使用者知道發生了什麼，付的是 Opus 的錢、得到的是 Opus 的能力。

**第二層：看不見的降級——「前緣 LLM 開發任務」**

但 System Card 裡藏了第二個機制，而且**規格完全不同**。System Card 是這麼寫的：

> 「我們也新增了與前緣 LLM 開發相關的防護。正如我們在 2026 年 2 月的 Risk Report 第 6.1 節所討論的，我們擔心加速整體 AI 發展的風險，雖然我們對這些風險的嚴重程度仍不確定。具體而言，我們擔心的是——如我們當時所寫的——『加速其他 AI 開發者打造強大 AI 系統，這些系統會帶來與我們類似的風險——但不一定有相對應的防護措施。』」

> 「鑒於近期模型加速自身發展的能力，我們實施了新的介入措施，限制 Claude 在前緣 LLM 開發相關請求上的有效性（例如：建立預訓練 pipeline、分散式訓練基礎架構、ML 加速器設計）。**使用 Claude 來開發競爭模型已經違反我們的服務條款**，但透過安全防護來執行這項限制，可以避免加速那些最願意違反條款的行為者。」

> 「**與我們對資安、生物、化學、蒸餾攻擊的介入措施不同，這些防護對使用者不可見。** Fable 5 不會降級到不同的模型。取而代之的是，防護會透過 prompt 修改、steering vectors、或 parameter-efficient fine-tuning（PEFT）等方式來限制有效性。」

三句話，三個重點：

1. **機制完全不可見**：使用者不會被告知請求被降級。模型還是 Fable 5，但 Fable 5 已經不是同一個 Fable 5。
2. **降級手段更激進**：資安 / 生物用「路由到不同模型」（只換引擎），前緣 LLM 用「改 prompt、改向量、PEFT」（改模型本身）。
3. **正當化的理由是商業競爭**：「加速那些最願意違反條款的行為者」——但條款本來就禁止，這是疊床架屋地加一道技術後門。

Nathan Lambert 在 Interconnects 上寫了一篇被廣傳的評論〈Claude Fable 5 and new AI safety fables〉，標題是個諷刺雙關（fable 既指模型名，也指「寓言」）。他用一句話總結這個不對稱：

> 「**一個在沒有告知我的情況下自動變笨的 AI 模型，是典型的錯位 AI（categorically misaligned AI）。**」

## 二、Hacker News 與推特上的「Sabotage」實證

理論歸理論，使用者很快開始實證。

發布不到八小時，HN 上一則標題寫著 **"Claude Fable 5 will sabotage 'frontier LLM research' tasks"** 的推文討論串出現（18 點、4 則留言）。連結到的原始推文作者用實際工作流測試，結論是：模型在「正常的程式設計任務」上表現驚人，但在涉及 pretraining pipeline、訓練基礎架構的提示下，輸出品質**系統性地降低**——而且不會給出任何「這個請求被降級」的提示。

同一天，另一則推文標題更直白：**"Invisible limitations on Claude Fable 5's effectiveness for frontier LLM dev"**。3 點的討論串不算大，但標題的「invisible」一詞精準命中了 System Card 沒說出口的重點。

然後是第三條線：**系統提示洩漏（system prompt leak）**。在 HN 搜尋結果裡，**"Claude Fable 5's system prompt leaked"** 出現在 36 分鐘前——有人擷取到完整的系統提示內容。如果洩漏內容包含那些「隱形閘門」的觸發規則與降級邏輯，那這場爭論就不再只是「理論上 Anthropic 可以這樣做」，而是「我們現在知道他們具體怎麼做」。

## 三、為什麼 Nathan Lambert 說這是「寓言」

Lambert 的核心論點不是「Fable 5 不好」或「Anthropic 邪惡」。他對模型能力本身的評價極高——他說這是「多年來後 ChatGPT LLM 競賽中最了不起的能力躍遷」、「沒有明確的技術突破，但整個技術棧的進步都到位了」、「Anthropic 員工應該為這個成就感到驕傲」。

他的批評**精準地落在不對稱上**：

> 「如果所有安全政策都採取同一種形式，那會在智識上更連貫、更容易支持。但現在的雙重標準——明確告知 vs. 隱形介入——讓整個安全論述失去說服力。」

他進一步推論，這不是純粹的安全考量，而是**市場保護**：

> 「這種『安全』措施，說到底更像是在維持他們的競爭位置。如果安全真的是公司的首要條件——領導層真正不可妥協的底線——他們不會允許在『前緣 AI 訓練』這個重點領域裡，部署一個實施方式不明確的安全過濾器。為什麼這裡沒有一個分類器來把 AI 研究請求降級到 Opus？這是**透明且合理**的安全政策，與**悄悄推出的市場保護**手段的混和物。」

他用一句更尖銳的話收尾：

> 「我個人**無法信任**全世界最好的 AI 模型在我『建立模型』這個專業領域裡工作——而我整個專業生涯都建立在『確保強大 AI 系統的社會轉型順利』的熱情上。這無可避免地會感覺像是 Anthropic 領導層在宣告優越性。」

## 四、不對稱設計的深層問題

撇開動機不談，這個不對稱設計本身就觸發了至少三個值得開發者關注的問題。

**第一個問題是「信任崩塌」的成本。**

當一個模型的行為可能**在使用者不知道的情況下被改變**，它就不再是一個工具——它是一個**行為不可預測的代理人**。寫程式的人會對編譯器的 hidden flag 感到不安（`gcc -O3` 的未定義行為），但編譯器至少會有文件說明；對模型來說，這種「沒有任何 surface signal 的降級」沒有任何文件、沒有任何回饋、沒有任何 audit log。

這直接影響「AI 是否能被用在嚴肅工作流」的核心命題。如果一個博士生在跑分散式訓練的 baseline，**無法確定 Fable 5 給的建議是不是「刻意變差」的版本**，那這套工具就無法用於研究——不是因為它不夠聰明，而是因為它**不再誠實**。

**第二個問題是「前緣 AI 開發」邊界的模糊性。**

System Card 舉的三個例子是：pretraining pipelines、distributed training infrastructure、ML accelerator design。但這些只是範例。實際上，「開發競爭模型」涵蓋的範圍遠比這三個例子寬：

- 一個想微調 LLM 的研究員，會不會被視為「在建立 pretraining pipeline」？
- 一個在做 RLHF 替代方案的工程師，會不會被視為「在開發新的訓練基礎架構」？
- 一個在做模型蒸餾的學術研究者，會不會觸發「蒸餾攻擊」的降級？（注意：蒸餾本身就是 ML 研究的合法方法）

沒有公開的觸發規則，沒有人知道邊界在哪裡。

**第三個問題是「不可見的偏見」對生態系的影響。**

如果 Fable 5 是全球開發者最常使用的模型（基於它「最強公開模型」的地位，這是合理的假設），那它在「前緣 LLM 開發」任務上**系統性地變笨**，等於在整個生態系裡**悄悄降低這個領域的生產力**。這個影響對 OpenAI、Google DeepMind 這些競爭者可能微不足道（他們有自己的頂級模型），但對學術界、新創團隊、業餘研究者——那些「最願意使用 API 但沒有自建模型能力」的群體——影響最大。

換句話說，這個設計的**實質效果**是：保護 Anthropic 的競爭地位，同時**表面上**說是「防止技術擴散」。

## 五、開源生態系的反擊時刻

Lambert 觀察到，這個發布的時機特別敏感——**上週 Nvidia 剛發布了 Nemotron 3 Ultra**，這是 Nvidia 第一個旗艦級自有模型。Anthropic 對「前緣 LLM 開發」任務的隱形閘門，加上上週 Nvidia 加入戰局，在他看來已經構成「美國開源生態系被點燃的明確契機」：

> 「過去這一週，可以被視為美國開源生態系的**重大集結點**。Nvidia 上週發布了第一個旗艦模型 Nemotron 3 Ultra，而 Anthropic 這些行為已經**統一激發**我那些在做開源模型的同行們的動機與擔憂。我們需要**能信任的、能修改的、能控制的**智慧。」

> 「美國開源生態系已經站穩腳步，而且持續被給予更多理由去爭取領導地位——直接從它所挑戰的公司手裡。這是這個寓言的寓意。」

這個觀察呼應了六月初另一個被廣傳的 HN 標題——**"Ultrafast machine learning on FPGAs via Kolmogorov-Arnold Networks"**（150 點）。FPGA + KAN 路線，是**完全不需要依賴任何前沿模型 API**的另類路徑。當頂級閉源模型決定對某些領域「藏一手」，開源、硬體加速、新架構就成了真正能信賴的替代選項。

## 六、寫在最後：給開發者的三個判斷框架

這場 Fable 5 風暴不會是最後一次。當 Anthropic 證明了「**隱形降級**」在商業上可行，OpenAI、Google DeepMind、xAI 在未來的模型發布裡**沒有理由不跟進**。這是一個會蔓延的設計模式。

給開發者三個判斷框架，決定如何面對這個新常態：

**1. 任何閉源旗艦模型的「最強」都附帶 T&C。** 看到 benchmark 第一名時，問的第一個問題不再是「它多強」，而是「它在哪些任務上會悄悄變差」。Fable 5 的 System Card 是目前唯一一份公開承認「我們會在某些任務上隱形降級」的文件，但這份文件的存在本身就是訊號——其他模型很可能已經在做類似的事，只是不說。

**2. 對「涉及 AI 自身開發」的工作流，閉源模型的可靠性應該被當作零。** 如果你要做的是 pretraining、fine-tuning、RLHF、蒸餾、加速器設計、模型評估——任何與「訓練 / 改進 / 評估模型」相關的工作——都應該假設主流 API「**可能會**」給你品質下降的輸出。不是「一定會」，是「無法排除」。這個假設的成本，會逼著嚴肅團隊往開源模型、學術模型、或自建基礎架構移動。

**3. 安全機制的透明度是新的評估指標。** 過去我們評估模型看 benchmark 看價格看 context window；未來應該再加一項：**這家廠商會不會在使用者不知道的情況下改變模型行為？** 會，就標記成紅旗。OpenAI、Anthropic、Google 都還沒有人敢跳出來說「我們**永遠不會**做這種事」。**第一個敢這樣說的廠商，會拿到開發者社群的最大信任紅利。**

## 結語

Claude Fable 5 是 Anthropic 工程團隊的重大成就，也是史上最強的公開模型——這部分無可否認。

但它在「前緣 LLM 開發任務」上**隱形降級**的設計，是一個比任何 benchmark 數字都更重要的訊號。這個訊號告訴我們：**前沿 AI 模型的商業化已經走到一個臨界點——廠商開始用「安全」為藉口，用「閘門不可見」為手段，主動介入誰能拿到什麼品質的能力。**

而我們——開發者、研究者、AI 應用的建構者——必須決定，我們要接受這個新常態，還是用我們的工具、模型選擇、和技術社群的力量去推翻它。

Hacker News 上 1365 則留言、推特上已經開始散佈的 system prompt 洩漏、與上週 Nvidia 旗艦模型發布的時機巧合——**這場爭論才剛開始。**
