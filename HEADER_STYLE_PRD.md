# Header 樣式改造 PRD

## 目標
根據截圖分析，重新設計 Header 和 Layout 樣式，使其符合 Stripe Blog 風格。

---

## 截圖分析

### 現況問題（從截圖觀察）

| 項目 | 現況 | 問題 |
|------|------|------|
| Header 背景 | 灰色 (#eeeeee) | 應該透明 |
| Logo | "Frank Lin" 文字 | 應該移除或簡化 |
| 導航位置 | 靠右 | 應該靠左 |
| 導航項目 | Blog, Portfolio, Home | 順序可能需要調整 |

| Page Title | "Articles" | 應該是 "Blog" |

---

## 風格規格（從截圖觀察）

### 1. Header 區域
- **背景**: 透明（無背景色）
- **高度**: 約 50-60px
- **內容寬度**: 最大 1200px，居中
- **導航**:
  - 位置：左側對齊
  - 項目：從左到右（Home, Blog, Portfolio）
  - 字體：14px，無襯線
  - 顏色：#697386
  - Hover：#1A1F36
- **分割線**: 底部 1px #E3E8EE

### 2. Page Title 區域
- **文字**: "Blog"（不是 "Articles"）
- **位置**: 左側對齊
- **字體大小**: 40px（桌面）/ 32px（手機）
- **字重**: 700 (bold)
- **顏色**: #1A1F36
- **間距**: 上 32px，下 24px

### 3. 整體 Layout
- **頁面背景**: #F7F9FC（淺灰）
- **內容區域**: 最大寬度 1200px
- **卡片**: 白色背景，圓角 8px，陰影

## 技術說明

### 根本原因
`_app.js` 中包含了錯誤的 CSS 語法（@tailwind 指令），導致 Tailwind 樣式無法正確編譯。

### 修復方向
1. 清理 `_app.js`，只保留 React 代碼
2. 確保 Tailwind 正確配置
3. 使用 Tailwind 類名實現設計
